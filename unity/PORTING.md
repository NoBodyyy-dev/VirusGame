# Порт VirusGame: Godot 4.7 → Unity 6 (URP)

Перенос кооп-хакинг-роглайта с Godot (GDScript) на Unity (C#). Godot-оригинал
живёт в `../project/` и остаётся рабочим; этот каталог — отдельная ветка `unity-port`.

## Как открыть

1. Unity Hub → Add → выбрать `unity/VirusGame` (Unity 6000.0.x, см. `ProjectVersion.txt`).
2. При первом открытии Unity сам подтянет пакеты из `Packages/manifest.json`
   (URP, Input System, TextMeshPro) и сгенерирует `Library/`, `.csproj`, `.meta`.
3. Создать URP-ассет (Assets → Create → Rendering → URP Asset) и назначить его в
   Project Settings → Graphics (окружение/пост-обработка — см. TODO).
4. Сцены (`.unity`) в репозиторий не входят — их собирают в редакторе (ниже).

### Известные ограничения текущего среза (не баги сборки)
- Рендер на встроенном конвейере: нет SSAO/SSR/bloom/объёмного тумана (это URP).
- Материалы на Standard: без триплана и без реалистичных шумовых текстур Godot.
- Построен только каркас Грида + обучающий этап 0; вход в сервер грузит сцену
  `Level`, которой пока нет (см. стадии ниже).

## Обновление 5: кооп v2 (рейд-синхронизация) + Steam

**Рейд-кооп** (`Core/NetSync.cs` — протокол, `World/Level.cs` — интеграция):

- **Директор рейда** — игрок с наименьшим id в узле: владеет тревогой,
  роботами и крюками; при его уходе роль мигрирует автоматически.
- **Общая тревога**: директор шлёт `RAS` (тревога/эвакуация/маска вносов/
  добыча, 2 Гц); события не-директора (задача −8, глушилка −12, удары,
  реанимация) идут поправками `RALD`.
- **Роботы**: директор целится во ВСЮ стаю (на охоте — сильнейший штамм по
  уровню эволюции, иначе ближайший), позиции 8 Гц (`RGP`); реплики сами
  засчитывают melee-урон по своему игроку; ЭМИ не-директора — `RSTN`.
- **Крюки**: позиции реплик `RHP`/`RHE`; зацеп жертва фиксирует сама и шлёт
  `RHC` — директор сматывает крюк.
- **Лут общий**: захват/отпуск `RLC` (гонка решается по меньшему id),
  позиция ноши `RLP`, бросок с вектором `RLT`, внос `RLD` (добыча — абсолют,
  эвакуация открывается у всех). Поздний вход догоняется маской в `RAS`.
- **Совместная переноска**: второй носильщик в 4 м ускоряет тяжёлый груз
  (×1.6 к фактору, до 0.92).
- **Общие арены**: сид кампании в снапшоте (`SNAP|…|seed`) +
  `GameState.ReseedCampaign` — узлы хоста и клиента дают одинаковые арены
  (раньше сиды были случайны на каждом клиенте!).
- Ловушки остались личными (система бьёт каждого вторженца отдельно) — это
  дизайн, не долг.

**Steam-кооп** (`Net/SteamNet.cs` — партиал NetManager под `STEAMWORKS_NET`):

- Пакет `com.rlabrecque.steamworks.net` (UPM git, manifest.json). Define
  ставит `UnityBuild.EnsureSteamDefine()` только если пакет реально
  установлен; без пакета игра собирается как раньше (кнопки Steam скрыты).
- Лобби: «КООП: STEAM — СОЗДАТЬ ЛОББИ» (публичное, фильтр `vg=1`) /
  «НАЙТИ ЛОББИ» (RequestLobbyList по фильтру) + приглашения через оверлей
  (GameLobbyJoinRequested). Аргументы `-steamhost` / `-steamjoin`.
- Транспорт: legacy P2P (`SendP2PPacket` reliable, канал 0), тот же
  строковый протокол NetSync — пир хранит steam-id вместо TCP-сокета,
  SendTo/Handle общие. Чтение — в Update главного потока.
- AppId 480 (Spacewar, тестовый): `steam_appid.txt` пишется рядом с exe
  при сборке и при первом запуске. Без запущенного Steam-клиента
  `SteamAPI.Init()` тихо падает → доступен только LAN.
- Шимы `_verify_full/SteamShims.cs`: весь Steam-путь компилируется офлайн.

## Обновление 8: пакет разнообразия — «каждый сервер — своя игра»

Ответ на главную жалобу плейтестов: «все компы одинаковые, после пятого
заражения скукота». Референсы: R.E.P.O. (хрупкий физический лут),
RV There Yet (общее дело и роли), Backrooms (уровень = правило, а не статы),
Keep Talking (асимметрия информации).

- **Архетипы серверов** (`GameData.ARCHETYPES`, `ArchForNode`): у узла Грида
  теперь есть ПРАВИЛО, а не только числа. Назначается детерминированно от
  сида узла (кооп и сейв дают то же; зона 0 чистая, ~25% узлов без твиста):
  ПОЧТОВЫЙ УЗЕЛ (спам-пустышки среди лута: ценники скрыты, фальшивку выдаёт
  глитч-дрожь, внос — тревога +6; spyware видит красную метку), ИГРОВОЙ
  СЕРВЕР (чит-гравитация + парящие платформы с жирным лутом), УМНЫЙ ДОМ
  (циклы темноты: запоминай зал, ловушки в темноте не летят), ЗЕРКАЛЬНЫЙ
  ПРОКСИ (зона выноса мигрирует запад↔восток; в коопе индекс площадки идёт
  в RAS), АРХИВНЫЙ МАССИВ (скан целостности: замри или маскируйся, иначе
  тревога +12 и залп), ПЛАТЁЖНЫЙ ШЛЮЗ (перепрыгиваемые лазерные гребёнки
  ходят по залу), ЛАБОРАТОРИЯ АВ (карантин: ловушек меньше, датчики зорче,
  глушения нет — чистый стелс; числа ужимаются в StartHack). Архетип виден
  в промпте узла в Гриде, в анимации входа и тостом на старте рейда.
- **Пул головоломок ×4** (`UI/Puzzles.cs` + роутер в `PuzzleUI`): к «Саймону»
  добавлены МАРШРУТ ПАКЕТА (поверни трубки, соедини вход с выходом; BFS-поток
  подсвечивает прогресс), ПОДБОР КЛЮЧА (мастермайнд по глифам: ●2 ○1;
  попытки кончились — ключ сменился) и КОММУТАЦИЯ (легенда-шифр, соедини
  порты; 3 ошибки — смена шифра). Тип двери стабилен (хеш ключа),
  15 пилонов Оракула циклятся по пулу. Общая оболочка — `PuzzleBase`.
- **Хрупкий лут** (T1+, ~26% файлов, цена ×1.4): стеклянный ореол;
  жёсткий удар после броска/падения режет цену ×0.72 (`FragileWatch` /
  OnCollisionEnter ≥7.5 м/с), на трети исходной цены — вдребезги и тревога
  +8 (пропажу видит вся стая: `RLG`). Аккуратная переноска против данков.
- **Горячие пакеты** (T2: 1, T3: 2; ◈30–42): греются в руках (~7.5с до
  перегрева), стынут на полу, бросок [F] сбрасывает 25 нагрева → оптимум —
  ЖИВАЯ ЦЕПОЧКА из игроков. Перегрев: пакет выпадает, −15% цены, тревога +7.
  Ядро пакета — шкала нагрева (жёлтый → алый), писк учащается.
- **Терминал оператора** (замена консоли глушения): [E] — тактический вид
  сверху (вторая камера), маркеры над роботами, [ЛКМ] пинг-столб всей стае
  (`ROP`), [R] стоп-кадр системы на 4с всем (`ROF`, кд 25с), фоновый рост
  тревоги глушится, пока сидишь. Оператор уязвим: удар/крюк выбивают из
  терминала. Полная асимметричная роль для больших стай.
- Протокол: `RAS` расширен `padIdx`, добавлены `RLG`/`ROP`/`ROF`.
- Тест-хуки смоука: `-arch <имя>` и `-hot` при прямом запуске рейда.
- SelfTest: детерминизм архетипов, тир-гейтинг, совпадение после ресида
  (кооп), карантин в конфиге рейда, горячие пакеты на T2.

## Обновление 9: адаптивный антивирус, мимики, кооп-смоук

- **Адаптивный АВ** (`GameState.avSeen/avCounter`, `AV_LEARN_AT=6`): система
  «учится» на приёмах, которыми игрок злоупотребляет между рейдами — данки,
  глушение оператора (каждые 10с), морф, фантом, шифрование. На пороге
  следующий рейд T1+ получает КОНТРМЕРУ (анонс на старте: «АВ ИЗУЧИЛ ВАС»):
  фильтр бросков (данк без бонуса, +4 тревоги), анти-глушение (×0.5),
  эвристика маскировки (морф вскрывается через 4с, +3 тревоги),
  трассировка фантома (2с вместо 5), горячий резерв (заморозка 1.8с).
  Счётчик выученного приёма сбрасывается — меняй стиль, и АВ забудет.
  Обучение живёт в сейве (`av.*`). Обучение персональное (не синкается) —
  «антивирус запомнил именно ТВОЙ почерк».
- **Лут-мимик** (T1+ ~10%, в почтовом узле 20%): файл-приманка, неотличимый
  от настоящего (spyware видит красную метку). Подбор — краб вцепляется в
  спину: −3 BW/с, замедление, нельзя брать лут; стряхивается [F] (кулдаун
  1.2с), +2 тревоги. Золотой куш мимиком не бывает. Стряхнутые — в сводке
  результатов.
- **Экран результатов**: архетип узла в строке итога + сводка неприятностей
  (разбито хрупких / перегревов / мимиков стряхнуто).
- **Метка архетипа в Гриде**: цветной маячок над сервером — маршрут стаи
  можно планировать издалека.
- **Кооп-смоук**: два инстанса `-host`/`-join 127.0.0.1` с раздельными
  `-logfile` — 30с обмена, 0 исключений с обеих сторон (после расширения
  протокола RAS/RLG/ROP/ROF).

## Технологические решения по стеку (обновление 5)

Запрошенный стек разобран по цене/выхлопу; сделано сейчас:

- **Вместо HDRP** — выжимаем URP: Bloom/ACES/виньетка (PostFx), Forward+
  под сотни огней, **ReflectionProbe** на арену (неон бликует на металле —
  бюджетная замена SSR), туман по темам. HDRP-миграция отложена сознательно:
  на процедурных примитивах она не добавит «реализма», зато удвоит
  требования к железу игроков Steam и сломает batchmode-конвейер сборки.
- **Вместо VFX Graph** — процедурные **ParticleSystem** (`Util/Fx.cs`):
  мотыльки данных в рейде/Гриде, вихрь зоны выноса. VFX Graph требует
  авторинга ассетов в GUI-редакторе — несовместимо с текущим циклом
  «код → batchmode → скриншоты игрой».
- **Сеть**: остаёмся на своём TCP LAN + Steam P2P (лобби уже работают).
  NGO+Relay — это полный переезд синхронизации; вернёмся к нему, если
  понадобится >4 игроков со сложной физикой на клиентах.
- **Input System (new), Localization, UniTask, DOTween** — отложены:
  ввод простой (WASD+8 клавиш), твины делаем кодом, строк пока мало.
  Всё это добавляется безболезненно позже.

Геймплей этого обновления: **мутаторы рейда** (ЖИРНЫЙ КУШ ×3, НЕВЕСОМОСТЬ,
СКОЛЬЗКИЙ ПОЛ, ДВОЙНОЙ ЛУТ/ТРЕВОГА, БЕШЕНЫЕ КРЮКИ — с анонсом на старте,
обучение Т0 всегда чистое), **данк** (любой лут в зоне выноса засчитывается,
влетевший броском — «ТРЁХОЧКОВЫЙ» +15%; кидайте напарнику через зал!).

## Обновление 7: стек по ТЗ, сейв-чекпойнт, кооп на 8

**Графический стек — решения по пунктам ТЗ** (и почему именно так):

- **HDRP — НЕ мигрируем.** Проект строит весь мир кодом на URP-материалах;
  миграция переписала бы Mats/PostFx/шейдеры целиком ради эффектов, которые
  на боксовой геометрии не окупаются, и сломала бы headless-сборку. Вместо
  этого взяли эквиваленты в URP: **SSAO** (renderer feature — объём на
  стыках геометрии), **Reflection Probe** в рейде (неон бликует на металле
  и глянце — задача «SSR» для интерьера-коробки), Volume-пост уже стоит
  (Bloom/ACES/виньетка). Тот же принцип, что в самом ТЗ про физику:
  «оставайся на том, что ИИ и конвейер знают лучше».
- **VFX Graph — не нужен**: его ассеты авторятся в GUI-редакторе, а наш мир
  процедурный. «Потоки данных» сделаны обычным `ParticleSystem`
  (`Util/DataFx.Stream`) — сотни частиц, не миллионы; стоят вдоль стен
  рейда и над порталом.
- **Shader Graph → рукописный HLSL**: `Assets/Shaders/HoloCode.shader` —
  голограмма «бегущего кода» (колонки глифов, сканлайн, глитч) на
  энерго-барьерах этапов и щите Оракула (по ТЗ: «полупрозрачная стена, на
  которой виден переливающийся код»). Пасс без LightMode → работает в URP
  и Built-in. Материал `Resources/mat_holo` тянет шейдер в билд.
- **Netcode for GameObjects / Relay / Lobby — отложено**: рабочий транспорт
  уже есть (TCP LAN + Steam P2P, протокол NetSync, кооп v2 с директором
  рейда). NGO — это переписывание сети ради той же функциональности;
  Relay/Lobby требуют аккаунт UGS. Вернуться, если понадобится
  серверная авторитарность или матчмейкинг без Steam.
- **Input System / Localization / UniTask / DOTween — отложено**: ввод
  работает (PuzzleUI/EvolutionUI уже блокируют движение), корутин в проекте
  мало и они простые, тексты русские в коде — локализация отдельным этапом.

**Сейв-чекпойнт** (жалоба «появляюсь с самого начала»): в `Serialize`
добавлены `seed=` (сид кампании — арены рейдов после перезапуска те же),
`cur=` (точка возврата: `GridWorld.SpawnPlayer` спавнит у последнего узла),
`failedn=` (проваленные узлы). **Моты data fragments одноразовые**: подбор
ставит флаг `mote:<i>` — он уходит в сейв И рассылается стае (SetFlag →
кооп), у напарников мот гаснет живьём.

**Кооп на 8 игроков**:

- **Скейлинг от размера стаи** (`GameState.packSize`, ставит NetManager):
  на каждого сверх первого — квота +40%, +1 файл, +ящик на двоих,
  +робот на каждых двух, ловушки на 15% чаще, тревога на 12% живее.
  Соло-баланс не тронут (packSize=1 — базовые числа).
- **Сейф (вес 3)** на T2+: в коопе поднимается ТОЛЬКО вдвоём (напарник в
  4 м), в соло — черепаший шаг 0.18; ransomware тащит чуть бодрее.
- **Консоль глушения** — роль оператора: пока игрок стоит у консоли и
  держит E, фоновый рост тревоги гасится (события всё равно поднимают).
  Работает через AlarmEvent → в коопе поправки уходят директору рейда.
- Роли на большой рейд: носильщики (сейф ×2), оператор глушения, бегунок
  полевых задач (двойной рубильник ×2, кабель), разведчик-spyware.

**Фикс**: небоскрёбы этапа 1 строились без коллайдеров — в здание можно
было зайти. Теперь `Build.Solid` (в Гриде всё имеет коллизию, по ТЗ).

## Верификация без Unity (историческая — теперь Unity есть)

- `unity/_verify` — ядро правил + **SelfTest** (`dotnet run`): 30+ поведенческих
